## Knowledge Items (KI) とは

**Knowledge Items（ナレッジアイテム、KI）**は、Antigravity独自の**永続的記憶システム**です。

従来のAIアシスタントは会話が終わると文脈をすべて失いますが、Antigravityは重要な知見をKIとして自動的に抽出・保存し、**未来の会話で再利用**できるようにします。

```
会話A: 「このプロジェクトではVitest使います」
       → KI自動生成: テストフレームワークの選択

会話B: （別の日）「テスト書いて」
       → KIを参照 → Vitestでテストを生成
       繰り返し説明する必要がない！
```

<Aside type="note">
Knowledge Items は Antigravity **固有** の機能です。Cursor や Claude Code には同等の自動永続記憶システムはありません。
</Aside>

## KI の内部構造

各KIは以下のファイル構造で保存されます：

```
~/.gemini/antigravity/knowledge/
├── knowledge.lock                      ← ロックファイル
└── <ki-id>/                            ← 各KIのディレクトリ
    ├── metadata.json                   ← サマリ・タイムスタンプ・参照元
    └── artifacts/                      ← 関連ファイル
        ├── documentation.md
        ├── code-example.ts
        └── ...
```

### metadata.json の内容

```json
{
  "summary": "このリポジトリではVitestを使用し、...",
  "created_at": "2026-03-15T10:30:00Z",
  "updated_at": "2026-03-28T14:20:00Z",
  "references": [
    {
      "conversation_id": "abc123-...",
      "source": "ユーザーの指示"
    }
  ]
}
```

## KI の自動生成フロー

<Steps>
1. **パターン検出**
   
   ユーザーとの会話中に、繰り返し参照されるパターンや重要な決定事項を検出

2. **知識の抽出**
   
   会話のコンテキストから、汎用的に再利用可能な知見を抽出

3. **KI の生成・更新**
   
   新しいKIを作成するか、既存のKIを更新。`metadata.json` にタイムスタンプと参照元を記録

4. **次の会話で活用**
   
   会話の開始時にKIサマリが提供され、エージェントは関連するKIを自動的に参照
</Steps>

## KI の活用ルール

Antigravityのエージェントは、以下の**優先順位**でコンテキストを探索します：

```
優先度: 高
  ↑  ① Knowledge Items（キュレーションされた知識）
  │  ② Conversation Logs（過去の会話ログ）  
  ↓  ③ Fresh Research（新規調査：Web検索等）
優先度: 低
```

### KI が特に効果を発揮するシナリオ

| シナリオ | 例 | KIの役割 |
|---|---|---|
| **「見た目は簡単」なタスク** | 「ログを追加して」 | リポジトリ固有のロギングパターンを提供 |
| **デバッグ** | 原因不明のエラー | 過去に遭遇した既知バグ・ゴッチャを参照 |
| **アーキテクチャ設計** | 「新機能を追加したい」 | 既存の類似パターン（プラグイン、キャッシュ等）を提示 |
| **複雑な統合作業** | CI/CDパイプライン | 過去のワークフロー例や設定パターンを参照 |

<Aside type="caution" title="重要：KIはスナップショットである">
KI は過去のある時点のスナップショットです。APIの変更、依存関係の更新、設定スキーマの変更により、KI内の情報が**古くなっている可能性があります**。

- 必ず現在のコードと照合してから使用する
- 非推奨の警告や欠落しているコンテキストを能動的に確認する
- KI は出発点であり、最終的な真実ではない
</Aside>

## KI の管理

### Agent Manager での確認

Antigravityの **Agent Manager** 画面で、蓄積されたKIを一覧・管理できます。

### 運用のベストプラクティス

1. **定期的なレビュー**  
   KIが古くなったパターンに依存していないか定期的に確認

2. **ワークスペースの戦略的な分離**  
   実験用と本番用でワークスペースを分けることで、コンテキストの汚染を防ぐ

3. **自然な蓄積を信じる**  
   初日からKIを大量に作ろうとしない。使い続ける中で自然に蓄積させるのが効果的

4. **フィードバックの活用**  
   エージェントが間違ったパターンを使ったら、正しいパターンを明示的に伝える → KIが更新される

## Conversation Logs との違い

| 概念 | Knowledge Items | Conversation Logs |
|---|---|---|
| **内容** | キュレーションされた知見 | 生の会話ログ |
| **生成** | 自動抽出 | 自動記録 |
| **構造** | `metadata.json` + artifacts | `overview.txt` |
| **用途** | パターン・決定の再利用 | 過去の文脈の詳細な参照 |
| **保存場所** | `~/.gemini/antigravity/knowledge/` | `~/.gemini/antigravity/brain/<id>/` |

### Conversation Logs の使い方

Conversation Logs は以下の場合に参照されます：

- ユーザーが過去の特定の会話に言及したとき
- KIでは不十分で、生の詳細が必要なとき
- 会話IDがわかっており、その内容が関連する可能性があるとき

```
~/.gemini/antigravity/brain/
├── <conversation-id-1>/
│   └── .system_generated/
│       └── logs/
│           └── overview.txt    ← 会話全体のトランスクリプト
├── <conversation-id-2>/
│   ├── implementation_plan.md  ← 計画Artifact
│   ├── task.md                 ← タスクArtifact  
│   ├── walkthrough.md          ← ウォークスルーArtifact
│   └── .system_generated/
│       └── logs/
└── ...
```

## 他ツールにおける類似概念

| ツール | 永続記憶 | 自動学習 |
|---|---|---|
| **Antigravity** | Knowledge Items（自動生成・管理） | ✅ 全自動 |
| **Cursor** | なし（Notepadsは手動メモ） | ❌ |
| **Claude Code** | `CLAUDE.md`（手動管理） | ▲ 半自動（セッション内のみ） |

<Aside type="tip">
Antigravityの最大の差別化要因の一つがこの Knowledge Items です。使い続けるほどエージェントの精度が上がる「学習するIDE」の核心です。
</Aside>

## この章のまとめ

- Knowledge Items はセッションを超える永続的な記憶システム
- 自動的に抽出・更新され、関連する会話で自動参照される
- KI → Conversation Logs → Fresh Research の優先順位を持つ
- KIはスナップショットなので、現在のコードとの照合が必須
- Conversation Logs は生の過去履歴、KIはキュレーションされた知見
